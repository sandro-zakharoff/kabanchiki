# Google Drive: пошаговая настройка хранилища фотографий

Фотографии задач, отчётов и аватарки могут храниться на вашем личном Google
Диске (папка `Kabanchiki/`) вместо Supabase. Приложение получает доступ
**только к файлам, которые создало само** (scope `drive.file`) — весь остальной
Диск ему не виден. Инструкция рассчитана на первый раз, с точными названиями
кнопок.

Всего три части: создать ключи в Google Cloud Console (~10 минут, один раз),
подключить их в Windows-программе, при желании перенести старые файлы.

---

## Часть 1. Google Cloud Console

### 1.1 Создать проект

1. Откройте <https://console.cloud.google.com/> и войдите в свой Google-аккаунт
   (тот, на чьём Диске будут храниться файлы).
2. Вверху слева, рядом с надписью «Google Cloud», нажмите на **выпадающий список
   проектов** → в открывшемся окне кнопка **«New project» / «Создать проект»**.
3. Название проекта: например `Kabanchiki` (любое). Нажмите **«Create» / «Создать»**.
4. Дождитесь уведомления и **выберите этот проект** в том же выпадающем списке.

### 1.2 Включить Google Drive API

1. В меню слева (☰): **«APIs & Services» → «Library»** («API и сервисы» → «Библиотека»).
2. В поиске наберите **Google Drive API**, откройте его.
3. Нажмите синюю кнопку **«Enable» / «Включить»**.

### 1.3 Настроить экран согласия (OAuth consent screen)

1. Меню ☰ → **«APIs & Services» → «OAuth consent screen»**
   (может называться «Google Auth Platform» → «Branding»).
2. Если попросит «Get started» — нажмите и заполните:
   - **App name:** `Kabanchiki`
   - **User support email:** ваш email
   - **Audience / User type:** **External** («Внешний»)
   - **Contact information:** ваш email
   - Согласитесь с политикой, нажмите **«Create» / «Создать»**.
3. Слева откройте раздел **«Audience»** и нажмите **«Publish app»**
   («Опубликовать приложение») → подтвердите **«Confirm»**.
   Это важно: без публикации Google отзывает доступ каждые 7 дней.
   Проходить проверку (verification) НЕ нужно — scope `drive.file`
   не является чувствительным.

### 1.4 Создать учётные данные (OAuth Client ID)

1. Меню ☰ → **«APIs & Services» → «Credentials»** («Учётные данные»).
2. Сверху **«+ Create credentials» / «Создать учётные данные»** → **«OAuth client ID»**.
3. **Application type:** **Desktop app** («Компьютерное приложение»).
4. Name: `Kabanchiki Desktop`. Нажмите **«Create»**.
5. Появится окно с двумя строками:
   - **Client ID** — вида `1234567890-abc123.apps.googleusercontent.com`
   - **Client secret** — вида `GOCSPX-…`
   Скопируйте обе (или скачайте JSON — значения те же внутри полей
   `client_id` и `client_secret`).

## Часть 2. Подключение в Windows-программе

1. Откройте Kabanchiki → **Налаштування** → группа **Google Drive**.
2. Вставьте **Client ID** и **Client Secret** в соответствующие поля.
3. Нажмите **«Підключити…»**. Откроется браузер:
   - выберите свой Google-аккаунт;
   - Google может показать предупреждение «Google hasn’t verified this app» —
     нажмите **«Continue» / «Продолжить»** (это ваше собственное приложение);
   - подтвердите доступ галочкой и **«Continue»**.
4. В браузере появится «Kabanchiki: Google Drive підключено ✔» — вернитесь
   в программу. Статус в настройках сменится на
   **«Підключено: ваш@gmail.com»**.
5. Проверьте кнопкой **«Перевірити з'єднання»** — тост покажет аккаунт и
   занятое место.
6. Переключите **«Зберігати нові фото у» → «Google Drive»**.
   С этого момента все новые фото со всех трёх приложений уходят на ваш Диск
   (папка `Kabanchiki/` с подпапками `tasks`, `proofs`, `avatars`).

Токены хранятся в диспетчере учётных данных Windows и на сервере семьи
(только для серверной функции загрузки с телефонов) — в репозиторий и к
клиентам они не попадают.

## Часть 3 (по желанию). Перенос старых файлов из Supabase

1. Убедитесь, что выполнены части 1–2 и вы залогинены в программе.
2. Откройте PowerShell в папке `desktop` репозитория и выполните:

   ```powershell
   .\.venv\Scripts\python.exe tools\migrate_to_drive.py --dry-run   # показать план
   .\.venv\Scripts\python.exe tools\migrate_to_drive.py             # перенести
   ```

3. Скрипт скопирует каждое фото на Диск, догенерирует миниатюры и перепишет
   ссылки в базе. **Из Supabase ничего не удаляется** — проверьте, что фото
   видны на всех трёх клиентах, после чего старые файлы можно удалить вручную
   в Supabase Studio (Storage → бакеты `task-photos`, `proof-photos`, `avatars`).

## Якщо на Диску з'явилися дві папки «Kabanchiki»

Так було в ранній версії: два шляхи завантаження (Windows-програма напряму та
серверна функція для телефонів) кешували ідентифікатори папок окремо й кожен
створював свою `Kabanchiki`. На роботу це не впливає — фото віддаються за
прямим id файлу, а не за шляхом. Починаючи з цієї версії обидва шляхи
використовують один спільний кеш у базі, тож нові папки більше не плодяться.

Щоб звести наявні дві папки в одну (файли переносяться за id — посилання в базі
не ламаються):

```powershell
.\.venv\Scripts\python.exe tools\merge_drive_folders.py --dry-run   # показати план
.\.venv\Scripts\python.exe tools\merge_drive_folders.py             # злити
```

Скрипт лишає одну `Kabanchiki`, переносить у неї всі файли з дублікатів і
відправляє порожні дублікати в Кошик Диска — після перевірки їх можна видалити
вручну.

## Как убедиться, что всё работает

- Создайте задачу с фото в Windows-программе → фото должно открыться в
  Mini App и на телефоне ребёнка.
- На Google Диске появится папка `Kabanchiki/tasks` с файлом.
- Отчёт ребёнка с фото с телефона → файл появится в `Kabanchiki/proofs`.

## Если что-то пошло не так

- **«redirect_uri_mismatch»** — тип клиента не «Desktop app». Создайте новый
  OAuth Client ID правильного типа.
- **«access_denied»** — на экране согласия вы не поставили галочку доступа.
- **Подключение слетает через неделю** — приложение не опубликовано:
  часть 1.3, «Publish app».
- **Фото не грузятся с телефона** — проверьте статус в Налаштування →
  Google Drive; при недоступности Диска загрузка автоматически уходит в
  Supabase (ничего не теряется).
- Отключить можно в любой момент кнопкой **«Відключити»** — новые фото снова
  пойдут в Supabase, старые ссылки на Диск продолжат работать.
