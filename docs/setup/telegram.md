# Telegram Mini App — налаштування

Керування родиною прямо з Telegram: ті самі акаунти батьків, що й у Windows-програмі,
миттєва синхронізація з десктопом у реальному часі. Хостинг — GitHub Pages (безкоштовно).

Архітектура:

```
Telegram → Mini App (GitHub Pages, telegram/) → Edge Function tg-auth
   → перевірка підпису initData секретом бота (лише на сервері)
   → сесія Supabase → ті самі RLS/RPC, що й у десктопі
```

Усі налаштування Telegram задаються у Windows-програмі (Налаштування → Telegram):
ім'я бота, посилання Mini App і **токен бота**. Токен — потужний секрет, тому він
зберігається у захищеній серверній таблиці, яку **не може прочитати жоден клієнт**
(навіть інший батько): його читає лише функція `tg-auth`. У програмі поле токена
приховане й не показує збережене значення — можна лише вписати новий.

---

## Крок 1. Створити бота у @BotFather

1. У Telegram відкрийте **@BotFather** → `/newbot`.
2. Вкажіть назву (напр. `Kabanchiki`) та **username** — має закінчуватися на `bot`
   (напр. `my_family_kabanchiki_bot`). Username знадобиться далі.
3. BotFather надішле **token** виду `1234567890:AAE...`.

## Крок 2. Прив'язати Mini App до кнопки меню

Після того, як сайт опубліковано (Крок 4), у @BotFather:

1. `/mybots` → оберіть бота → **Bot Settings → Menu Button → Configure Menu Button**.
2. Введіть URL Mini App:
   HTTPS-адресу GitHub Pages цього репозиторію
3. Текст кнопки — напр. «Керування».

(Альтернатива: `/newapp` → прив'язати той самий URL як іменований Mini App.)

## Крок 3. Налаштувати ім'я бота у Windows-програмі

Kabanchiki (десктоп) → **Налаштування → Telegram**:

- **Ім'я бота** — username з Кроку 1 (без `@`).
- **Посилання на Mini App** — вже заповнено:
  URL опублікованої сторінки цього репозиторію
- Натисніть **Зберегти налаштування Telegram**.
- **Токен бота** — вставте token з Кроку 1 у поле «Токен бота» → **Зберегти**.
  Індикатор покаже «Токен встановлено». Значення назад не показується (це нормально).

## Крок 4. Публікація Mini App (GitHub Pages)

Основний репозиторій містить Mini App у каталозі `telegram/`. Workflow
`Deploy Telegram Mini App` публікує цей каталог у GitHub Pages з єдиного репозиторію.

Mini App — публічна сторінка за задумом (Telegram відкриває її будь-кому), секретів у ній
немає (лише публікований anon-ключ). Після push оновлення публікуються автоматично.

## Крок 5. Прив'язати свій Telegram (кожен батько)

1. Windows → **Налаштування → Telegram → «Прив'язати мій Telegram»**.
2. Програма покаже одноразове посилання (діє 15 хв). Відкрийте його на телефоні,
   де встановлено ваш Telegram, і натисніть **Start**.
3. Готово — Mini App відкривається під вашим акаунтом. Наступні відкриття — автоматично.

Другий батько (власник) додається у Windows (Налаштування → Власники), після чого
так само прив'язує свій Telegram.

---

## Крок 6. Сповіщення бота (з версії 1.8.0)

Бот сам надсилає власникам сповіщення про дії виконавців (виконав/відправив на
перевірку завдання, попросив вивід, прострочений дедлайн) з кнопками швидких дій
(«Прийняти» / «На доробку» / «Відхилити», «Схвалити» / «Відхилити»). Натискання
кнопки одразу змінює статус у системі та у журналі — від імені того власника,
який натиснув.

**Нічого налаштовувати вручну не треба:** вебхук бота реєструється автоматично,
коли ви зберігаєте налаштування Telegram у Windows-програмі (Крок 3). Якщо змінили
токен — просто збережіть ще раз, вебхук перевстановиться.

Технічно: подія виконавця → таблиця `tg_outbox` → Edge Function `tg-notify`
(`sendMessage` кожному прив'язаному активному власнику). Кнопки обробляє
Edge Function `tg-bot` (вебхук бота, захищений `secret_token`). Невдалі відправки
повторюються (pg_cron, до 5 разів) і, якщо так і не дійшли, лишають запис у журналі.

## Перевірка

- Відкрийте бота в Telegram → кнопка меню → Mini App має показати ваші завдання/роботи.
- Дія в Telegram (напр. «Прийняти» завдання) миттєво відображається у Windows і навпаки —
  обидва клієнти підписані на Realtime.
- Нехай виконавець відправить завдання на перевірку — вам у чат з ботом має прийти
  повідомлення з кнопками. Натисніть «Прийняти» — статус зміниться всюди.
